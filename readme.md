# Итоговый проект | Алгоритмы и структуры данных

## Структура проекта

Проект состоит из четырёх основных классов, расположенных в src/programs:

### 1. `GeneratePresetImpl.java`
- **Назначение:** отвечает за генерацию пресета армии противника.
- **Реализован жадный алгоритм**
- **Сложность: O(n * m)**, где n - число типов, m - максимальное число юнитов

### 2. `SimulateBattleImpl.java`
- **Назначение:** отвечает за осуществление симуляции боя.
- **Сложность: O(n² log n)**, где n - число типов, m - максимальное число юнитов
- - Внешний цикл (while): O(n)
- - getAllLivingUnits(): O(n)
- - Сортировка (sort): O(n log n)
- - Внутренний цикл (for): O(n) итераций по юнитам
- - O(n) × [O(n) + O(n log n) + O(n)] = O(n² log n)

### 3. `SuitableForAttackUnitsFinderImpl.java`
- **Назначение:** отвечает за создание перечня подходящих для атаки юнитов.
- **O(n)**, где n — общее количество юнитов во всех рядах.
- - Внешний цикл по 3 рядам → O(3) = O(1)
- - Для каждого ряда:
- - Сбор занятых Y-координат: O(m), где m — юнитов в ряду
- - Проверка каждого юнита: O(m)
- - В сумме: O(3 × 2m) = O(6m) = O(n)

### 4. `UnitTargetPathFinderImpl.java`
- **Назначение:** отвечает за поиск наикратчайшего пути между атакующим и атакуемым юнитом.
- **O(W·H·log(W·H))**, где W и H — ширина и высота поля.
- - Посещение узлов: для сетки W×H: O(W·H) узлов
- - Обработка каждого узла: O(log n), где n — количество элементов в очереди
- - В худшем случае n = W·H
- - Итого: O(W·H × log(W·H))
